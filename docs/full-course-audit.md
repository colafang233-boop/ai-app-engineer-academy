## 3. 课程结构审查

### 3.1 连续学习路径

完整课程必须满足：

- 7 个唯一专栏 ID；
- 108 个唯一课程 ID 和连续编号 `1–108`；
- 7 个唯一考试 ID；
- 每个专栏只依赖前一专栏考试；
- 每课保持 `prediction → simulator → content → quiz` 四阶段；
- 正式入口默认顺序解锁。

新增 `tools/test_release_defaults.mjs` 将这些规则固化为发布门禁。

### 3.2 教学颗粒度

当前专栏分布：

| 专栏 | 课程数 | 审查判断 |
|---|---:|---|
| 基础模型请求 | 4 | 紧凑，适合作为 Java 开发者入口 |
| Prompt 与输出契约 | 5 | 概念链清晰，成果连续 |
| LangChain v1 | 12 | 覆盖当前高层 Agent 主线，不与 LangGraph 自定义拓扑混淆 |
| RAG 工程 | 17 | 评价先行、框架中立，避免把数据库或模型当唯一答案 |
| LangGraph / Agentic RAG | 20 | 从适用性到持久化、人工介入和生产发布，前置链完整 |
| MCP | 25 | 覆盖面完整，但协议即将大版本切换，时效风险最高 |
| 企业生产交付 | 25 | 统一场景有效，避免变成零散企业术语清单 |

第二轮需要重点检查 17–25 课长专栏内部是否存在：

- 可以合并的重复结论；
- 术语第一次出现早于解释；
- 学员在一课内承担过多新概念；
- 交互只是在选择预设正确答案，没有形成可迁移判断。

## 4. 内容正确性与版本审查

### 4.1 已核对的稳定基线

截至 2026-07-23，课程标注的以下版本仍与当前公开包一致：

- `langchain@1.5.3`
- `@langchain/core@1.2.3`
- `@langchain/langgraph@1.4.8`
- `langsmith@0.8.3`
- `@modelcontextprotocol/sdk@1.29.0`

课程采用“基线日期 + 版本 + 官方链接”的方式是正确的。后续不应自动追求每个 patch 都升级，而应先执行：

```text
Changelog / Migration Review
→ 代码示例兼容检查
→ 交互与题库检查
→ 浏览器完整旅程
→ 决定是否更新教学基线
```

### 4.2 MCP 专项风险

MCP 官方当前正式协议仍为 `2025-11-25`。`2026-07-28` Release Candidate 已锁定并计划在 2026-07-28 发布正式版，主要变化包括：

- 协议核心转为无状态；
- 移除初始化握手和协议级 Session；
- Extensions 成为正式机制；
- Tasks 转为 Extension 并调整生命周期；
- Roots、Sampling、Logging 进入弃用流程；
- 授权、请求头、缓存和 Trace Context 发生变化。

因此，当前专栏以 `2025-11-25 + SDK 1.29.0` 为生产主线仍然合理，但只能视为“发布前最后一个旧协议基线”。2026-07-28 后必须建立单独迁移 PR，不能只改版本号。

## 5. 交互与产品体验审查

### 已确认优点

- 所有专栏复用同一 CourseRuntime 教学语法；
- LangChain、RAG、LangGraph、MCP、企业交付均有专属实验装置；
- 答错可以重试，浏览器旅程已覆盖主要状态；

- 项目成果贯穿前后课程；
- PC 页面宽度、标题折行和 390px 页面级溢出已有自动回归。

### 下一轮重点

1. **移动端可读性**：不仅检查 `scrollWidth`，还要检查正文最小字号、按钮目标尺寸、sticky 导航、长表格和代码块。
2. **键盘与焦点**：抽屉打开后焦点捕获、Esc 关闭、关闭后焦点归还、选项键盘操作。
3. **长课程导航**：25 课专栏和超长实验页需要当前位置、阶段目录或快速返回顶部。
4. **反馈质量**：错误反馈是否真正解释原因，而不是只提示“错误”。
5. **真实运行边界**：明确哪些实验是确定性教学模拟，哪些是实际 TypeScript / Protocol / API 执行。

## 6. 数据、进度与恢复审查

当前 ProgressStore 和 ArtifactStore 使用 localStorage。优点是零后端、即时保存；风险是：

- 浏览器清理数据后 108 课进度全部丢失；
- 写入失败和配额超限没有用户可见反馈；
- 无法跨设备继续；
- 无法导出完整毕业作品集；
- 课程结构升级时没有显式数据迁移版本。

建议第二轮至少实现：

- JSON 导出/导入；
- 数据 Schema Version；
- localStorage 写入异常处理；
- 课程版本与成果版本写入元数据；
- 导入时的完整性校验和冲突策略。
